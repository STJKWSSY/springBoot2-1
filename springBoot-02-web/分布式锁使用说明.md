# Redis 分布式锁使用说明

## 一、添加的依赖

在 `pom.xml` 中已添加 Redisson 依赖：

```xml
<!-- Redisson - 分布式锁 -->
<dependency>
    <groupId>org.redisson</groupId>
    <artifactId>redisson-spring-boot-starter</artifactId>
    <version>3.17.0</version>
</dependency>
```

## 三、核心 API

### RedissonLockService 方法

| 方法 | 说明 | 参数 | 返回值 |
|------|------|------|--------|
| `tryLock(key, leaseTime)` | 尝试获取锁（非阻塞） | key, 持有时间 | boolean |
| `tryLock(key, waitTime, leaseTime)` | 尝试获取锁（带等待） | key, 等待时间，持有时间 | boolean |
| `lock(key)` | 获取锁（阻塞） | key | void |
| `lock(key, leaseTime)` | 获取锁（带超时） | key, 持有时间 | void |
| `unlock(key)` | 释放锁 | key | void |
| `isLocked(key)` | 检查锁状态 | key | boolean |

## 四、使用示例

### 4.1 基本用法

```java
@Service
public class OrderService {

    @Autowired
    private RedissonLockService lockService;

    public void doSomething(String userId) {
        String lockKey = "order:lock:" + userId;

        // 尝试获取锁，等待 3 秒，持有 10 秒自动释放
        boolean isLocked = lockService.tryLock(lockKey, 3, 10);

        if (!isLocked) {
            throw new RuntimeException("获取锁失败");
        }

        try {
            // 执行业务逻辑
            // ...
        } finally {
            // 释放锁
            lockService.unlock(lockKey);
        }
    }
}
```

### 4.2 秒杀场景

```java
@Service
public class SeckillService {

    @Autowired
    private RedissonLockService lockService;

    public String seckill(String userId, Long productId) {
        String lockKey = "seckill:lock:" + productId;

        boolean isLocked = lockService.tryLock(lockKey, 3, 10);
        if (!isLocked) {
            return "系统繁忙";
        }

        try {
            // 检查库存
            // 扣减库存
            // 创建订单
            return "秒杀成功";
        } finally {
            lockService.unlock(lockKey);
        }
    }
}
```

## 六、Redisson 优势

### 相比手动实现的优势：

1. **自动续期**（Watch Dog 机制）
   - 业务未执行完时自动延长锁的过期时间
   - 避免业务未结束锁就过期

2. **可重入**
   - 同一线程可以多次获取同一把锁

3. **公平锁/非公平锁**
   - 支持 FIFO 公平锁

4. **联锁（MultiLock）**
   - 支持 Redis 集群部署

5. **红锁（RedLock）**
   - 更安全的分布式锁实现

## 七、注意事项

### 7.1 锁的粒度

```java
// 锁粒度太粗 - 并发度低
String lockKey = "order:global";

// 锁粒度合适 - 按用户维度
String lockKey = "order:user:" + userId;

// 锁粒度合适 - 按商品维度
String lockKey = "seckill:product:" + productId;
```

### 7.2 必须释放锁

```java
// 错误示例 - 可能忘记释放锁
lockService.lock(key);
doSomething();  // 如果抛出异常，锁不会释放
lockService.unlock(key);

// 正确示例
boolean isLocked = lockService.tryLock(key, waitTime, leaseTime);
if (isLocked) {
    try {
        doSomething();
    } finally {
        lockService.unlock(key);
    }
}
```

### 7.3 持有时间设置

```java
// 持有时间应该大于业务执行的最大时间
// 如果业务时间不确定，使用 Redisson 的 Watch Dog 机制
lockService.lock(key); // 默认 30 秒，会自动续期
```

## 八、性能优化建议

1. **减少锁的持有时间**
   - 将不相关的操作移到锁外

2. **使用细粒度锁**
   - 按业务维度拆分锁

3. **避免数据库长事务**
   - 在锁内只执行必要的数据库操作

4. **设置合理的超时时间**
   - 避免死锁和资源浪费
