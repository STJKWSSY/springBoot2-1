# Spring Boot 拦截器使用说明

## 一、什么是拦截器

拦截器（Interceptor）是 Spring MVC 提供的一种机制，可以在请求到达 Controller 之前、Controller 处理请求之后、视图渲染之后执行特定的逻辑。

### 拦截器的执行流程

```
请求 → preHandle() → Controller → postHandle() → 视图渲染 → afterCompletion()
         ↓              ↓            ↓              ↓                    ↓
      前置处理      业务处理     后置处理      渲染完成            全部完成
```

## 二、项目中的拦截器实现

### 2.1 拦截器类：LoginCheckInterceptor

**文件位置：** `src/main/java/com/jiangnan/www/springboot02web/demos/web/interceptor/LoginCheckInterceptor.java`

```java
@Component
public class LoginCheckInterceptor implements HandlerInterceptor {

    // 1. 请求到达 Controller 之前执行
    @Override
    public boolean preHandle(HttpServletRequest request,
                            HttpServletResponse response,
                            Object handler) throws Exception {
        // 检查 session 中是否有 loginUser
        String loginUser = (String) request.getSession().getAttribute("loginUser");

        if (loginUser != null) {
            return true; // 已登录，放行
        }

        // 未登录，重定向到登录页
        response.sendRedirect("/login");
        return false; // 拦截请求
    }

    // 2. Controller 处理完后执行（视图渲染前）
    @Override
    public void postHandle(HttpServletRequest request,
                          HttpServletResponse response,
                          Object handler,
                          ModelAndView modelAndView) throws Exception {
        // 可用于日志记录、性能监控等
    }

    // 3. 整个请求完成后执行（视图渲染后）
    @Override
    public void afterCompletion(HttpServletRequest request,
                               HttpServletResponse response,
                               Object handler,
                               Exception ex) throws Exception {
        // 可用于资源清理
    }
}
```

### 2.2 配置类：WebConfig

**文件位置：** `src/main/java/com/jiangnan/www/springboot02web/demos/web/config/WebConfig.java`

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Autowired
    private LoginCheckInterceptor loginCheckInterceptor;

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(loginCheckInterceptor)
                // 拦截所有路径
                .addPathPatterns("/**")
                // 排除不需要拦截的路径
                .excludePathPatterns(
                        "/login",        // 登录页面
                        "/doLogin",      // 登录接口
                        "/",             // 首页
                        "/index.html",   // 首页文件
                        "/res/**",       // 静态资源
                        "/static/**",    // 静态资源
                        "/*.ico",        // 图标文件
                        "/*.png",        // 图片文件
                        "/*.jpg"
                );
    }
}
```

## 三、使用方法

### 3.1 创建拦截器步骤

1. **创建拦截器类**：实现 `HandlerInterceptor` 接口
2. **添加注解**：使用 `@Component` 注解交给 Spring 管理
3. **重写方法**：根据需要重写 `preHandle`、`postHandle`、`afterCompletion`
4. **配置拦截规则**：在配置类中使用 `@Configuration` + `WebMvcConfigurer`

### 3.2 配置拦截路径

```java
// 单个路径
.addPathPatterns("/api/**")

// 多个路径
.addPathPatterns("/api/**", "/admin/**")

// 排除特定路径
.excludePathPatterns("/public/**", "/login")
```

## 四、测试方法

### 4.1 启动项目

```bash
mvn spring-boot:run
```

### 4.2 测试流程

1. **访问受保护的页面**（未登录）：
   ```
   GET http://localhost:8080/main
   ```
   预期结果：重定向到 `/login`

2. **访问登录页**：
   ```
   GET http://localhost:8080/login
   ```
   预期结果：显示登录页面

3. **执行登录**：
   ```
   POST http://localhost:8080/doLogin

   ```

4. **再次访问受保护的页面**（已登录）：
   ```
   GET http://localhost:8080/main
   ```
   预期结果：正常访问

## 五、拦截器方法说明

| 方法 | 执行时机 | 返回值 | 用途 |
|------|----------|--------|------|
| `preHandle` | Controller 之前 | boolean：true 放行，false 拦截 | 权限检查、日志记录 |
| `postHandle` | Controller 之后，视图渲染前 | void | 修改 ModelAndView |
| `afterCompletion` | 视图渲染后 | void | 资源清理、异常处理 |

## 六、常见应用场景

### 6.1 登录检查
```java
if (session.getAttribute("user") == null) {
    response.sendRedirect("/login");
    return false;
}
return true;
```

### 6.2 权限检查
```java
if (!user.hasPermission(request.getRequestURI())) {
    response.setStatus(403);
    return false;
}
return true;
```

### 6.3 日志记录
```java
long startTime = System.currentTimeMillis();
request.setAttribute("startTime", startTime);

// afterCompletion 中
Long startTime = (Long) request.getAttribute("startTime");
long cost = System.currentTimeMillis() - startTime;
logger.info("请求耗时：" + cost + "ms");
```

### 6.4 统一数据处理
```java
// postHandle 中
if (modelAndView != null) {
    modelAndView.addObject("appName", "My Application");
    modelAndView.addObject("version", "1.0.0");
}
```

## 七、注意事项

1. **循环重定向问题**：登录接口和登录页必须排除，否则会无限重定向
2. **静态资源**：css、js、图片等静态资源建议排除，提高性能
3. **执行顺序**：多个拦截器时，按添加顺序执行
4. **异常处理**：`afterCompletion` 中可以获取异常信息进行处理
